🤝 GitHub Workflow

→Feature branching: No direct commits to main. Each member works on feature/page-name.
→Pull Requests: All code reviewed by at least one other engineer before merging.
→Issue tracking: GitHub Issues used to assign specific pages and tasks per member.
→Environment safety: API keys stored in .env and excluded via .gitignore.
